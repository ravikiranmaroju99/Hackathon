# OpsAI Assistant Overview

## What is OpsAI Assistant?

OpsAI Assistant is an enterprise IT operations support chatbot created to help cloud operations, Kubernetes, DevOps, SRE, infrastructure, service desk and application support teams.

It uses approved operational documents and runbooks stored in an Amazon Bedrock Knowledge Base.

## Current MVP Capabilities

The current AWS MVP can:

1. Answer questions using approved runbooks.
2. Support Kubernetes, RKE2 and Jenkins troubleshooting.
3. Provide easy numbered troubleshooting steps.
4. provide useful validation commands.
5. Maintain conversation context within the same chat session.
6. Show previous conversations in browser chat history.
7. Warn users that production changes require human approval.
8. Be accessed through a browser-based chatbot interface.

## Benefits of OpsAI Assistant

### 1. Faster Troubleshooting

OpsAI Assistant quickly searches approved operational runbooks and provides relevant troubleshooting steps. This can reduce the time spent manually searching documents.

### 2. Consistent Support

The assistant provides standardized guidance from approved sources. This helps different support engineers follow the same troubleshooting process.

### 3. 24x7 Self-Service Access

Users can access operational guidance at any time through the chatbot interface, subject to AWS service availability.

### 4. Knowledge Sharing

Important operational knowledge can be made available to support teams, new employees and project members through a common chatbot.

### 5. Faster Employee Onboarding

New team members can use OpsAI Assistant to understand approved procedures, troubleshooting commands and validation steps.

### 6. Reduced Dependency on Individual Experts

Operational knowledge is stored in an approved central Knowledge Base instead of depending only on the availability of specific engineers.

### 7. Human Approval for Critical Changes

OpsAI Assistant must request human approval before recommending or performing restarts, reboots, deletions, scaling, patching or production changes.

### 8. Validation After Troubleshooting

The assistant provides validation steps so that users can confirm whether the issue is resolved and services are operating correctly.

### 9. Reusable Across Projects

The solution can be reused for cloud operations, Kubernetes, DevOps, infrastructure and application-support requirements across different delivery teams and clients.

### 10. Future Integration Possibilities

Future versions can integrate with approved APIs for Jira, Jenkins, Kubernetes, monitoring tools, email and automation platforms.

These integrations are not enabled for automatic production actions in the current MVP.

## Security and Governance

The current MVP:

- Uses approved Knowledge Base documents.
- Does not automatically execute production changes.
- Requires human approval for critical actions.
- Uses IAM permissions to control AWS access.
- Stores application logs in Amazon CloudWatch.
- Provides validation steps after troubleshooting.

## Current MVP Architecture

The request flow is:

User browser interface  
→ Amazon Amplify Hosting  
→ Amazon API Gateway  
→ AWS Lambda  
→ Amazon Bedrock Agent  
→ Amazon Bedrock Knowledge Base  
→ Approved runbooks stored in Amazon S3

Step 18B — Sync the Updated Knowledge Base

18B with this corrected block

Paste the complete block below. It will overwrite the old function in the current CloudShell session.

sync_opsai_knowledge_base() {
  local PROJECT_DIR="$HOME/opsai-assistant-aws"
  local ENV_FILE="$PROJECT_DIR/.env"
  local KB_NAME="opsai-assistant-kb"

  local ACCOUNT_ID
  local KB_ID
  local KB_STATUS
  local DATA_SOURCE_ID
  local DATA_SOURCE_NAME
  local DATA_SOURCE_BUCKET_ARN
  local EXPECTED_BUCKET_ARN
  local ACTIVE_JOB_ID
  local INGESTION_JOB_ID
  local STATUS

  cd "$PROJECT_DIR" || {
    echo "ERROR: Project directory was not found."
    return 1
  }

  if [[ ! -f "$ENV_FILE" ]]; then
    echo "ERROR: $ENV_FILE was not found."
    return 1
  fi

  set -a
  source "$ENV_FILE"
  set +a

  AWS_REGION="${AWS_REGION:-us-east-1}"
  AWS_DEFAULT_REGION="${AWS_DEFAULT_REGION:-$AWS_REGION}"

  export AWS_REGION
  export AWS_DEFAULT_REGION

  echo "============================================================"
  echo " OpsAI Knowledge Base Synchronization"
  echo "============================================================"
  echo
  echo "Region: $AWS_REGION"

  # Automatically create the expected bucket name when it is missing.
  if [[ -z "${OPSAI_BUCKET:-}" ]]; then
    ACCOUNT_ID=$(aws sts get-caller-identity \
      --query Account \
      --output text) || return 1

    OPSAI_BUCKET="opsai-assistant-kb-${ACCOUNT_ID}-${AWS_REGION}"
  fi

  echo "S3 bucket: $OPSAI_BUCKET"

  if ! aws s3api head-bucket \
    --bucket "$OPSAI_BUCKET" \
    >/dev/null 2>&1
  then
    echo "ERROR: Bucket was not found or cannot be accessed:"
    echo "  $OPSAI_BUCKET"
    return 1
  fi

  echo
  echo "Knowledge documents currently stored in S3:"

  aws s3 ls \
    "s3://$OPSAI_BUCKET/knowledge/" \
    --recursive || return 1

  echo
  echo "Searching for the active Knowledge Base..."

  # Keep the complete JMESPath query on one line.
  KB_ID=$(aws bedrock-agent list-knowledge-bases \
    --region "$AWS_REGION" \
    --query "knowledgeBaseSummaries[?name=='$KB_NAME' && status=='ACTIVE'].knowledgeBaseId | [0]" \
    --output text) || return 1

  if [[ -z "$KB_ID" || "$KB_ID" == "None" ]]; then
    echo "ERROR: Active Knowledge Base $KB_NAME was not found."
    echo
    echo "Available Knowledge Bases:"

    aws bedrock-agent list-knowledge-bases \
      --region "$AWS_REGION" \
      --query 'knowledgeBaseSummaries[].{Name:name,ID:knowledgeBaseId,Status:status}' \
      --output table

    return 1
  fi

  KB_STATUS=$(aws bedrock-agent get-knowledge-base \
    --knowledge-base-id "$KB_ID" \
    --region "$AWS_REGION" \
    --query 'knowledgeBase.status' \
    --output text) || return 1

  echo "Knowledge Base name: $KB_NAME"
  echo "Knowledge Base ID: $KB_ID"
  echo "Knowledge Base status: $KB_STATUS"

  echo
  echo "Searching for an available S3 data source..."

  # Select the available data source without depending on its name.
  DATA_SOURCE_ID=$(aws bedrock-agent list-data-sources \
    --knowledge-base-id "$KB_ID" \
    --region "$AWS_REGION" \
    --query "dataSourceSummaries[?status=='AVAILABLE'].dataSourceId | [0]" \
    --output text) || return 1

  if [[ -z "$DATA_SOURCE_ID" || "$DATA_SOURCE_ID" == "None" ]]; then
    echo "ERROR: No AVAILABLE data source was found."
    echo
    echo "Configured data sources:"

    aws bedrock-agent list-data-sources \
      --knowledge-base-id "$KB_ID" \
      --region "$AWS_REGION" \
      --query 'dataSourceSummaries[].{Name:name,ID:dataSourceId,Status:status}' \
      --output table

    return 1
  fi

  DATA_SOURCE_NAME=$(aws bedrock-agent get-data-source \
    --knowledge-base-id "$KB_ID" \
    --data-source-id "$DATA_SOURCE_ID" \
    --region "$AWS_REGION" \
    --query 'dataSource.name' \
    --output text) || return 1

  DATA_SOURCE_BUCKET_ARN=$(aws bedrock-agent get-data-source \
    --knowledge-base-id "$KB_ID" \
    --data-source-id "$DATA_SOURCE_ID" \
    --region "$AWS_REGION" \
    --query 'dataSource.dataSourceConfiguration.s3Configuration.bucketArn' \
    --output text) || return 1

  EXPECTED_BUCKET_ARN="arn:aws:s3:::$OPSAI_BUCKET"

  echo "Data Source name: $DATA_SOURCE_NAME"
  echo "Data Source ID: $DATA_SOURCE_ID"
  echo "Data Source bucket: $DATA_SOURCE_BUCKET_ARN"

  if [[ "$DATA_SOURCE_BUCKET_ARN" != "$EXPECTED_BUCKET_ARN" ]]; then
    echo "ERROR: The selected data source uses a different S3 bucket."
    echo "Expected: $EXPECTED_BUCKET_ARN"
    echo "Actual:   $DATA_SOURCE_BUCKET_ARN"
    return 1
  fi

  update_env_value() {
    local KEY="$1"
    local VALUE="$2"

    if grep -q "^${KEY}=" "$ENV_FILE"; then
      sed -i \
        "s|^${KEY}=.*|${KEY}=${VALUE}|" \
        "$ENV_FILE"
    else
      printf '%s=%s\n' \
        "$KEY" \
        "$VALUE" \
        >> "$ENV_FILE"
    fi
  }

  update_env_value "AWS_REGION" "$AWS_REGION"
  update_env_value "AWS_DEFAULT_REGION" "$AWS_DEFAULT_REGION"
  update_env_value "OPSAI_BUCKET" "$OPSAI_BUCKET"
  update_env_value "BEDROCK_KB_ID" "$KB_ID"
  update_env_value "BEDROCK_DATA_SOURCE_ID" "$DATA_SOURCE_ID"
  update_env_value "BEDROCK_DATA_SOURCE_NAME" "$DATA_SOURCE_NAME"

  chmod 600 "$ENV_FILE"

  export BEDROCK_KB_ID="$KB_ID"
  export BEDROCK_DATA_SOURCE_ID="$DATA_SOURCE_ID"
  export BEDROCK_DATA_SOURCE_NAME="$DATA_SOURCE_NAME"

  echo
  echo "Checking for an ingestion job already in progress..."

  ACTIVE_JOB_ID=$(aws bedrock-agent list-ingestion-jobs \
    --knowledge-base-id "$KB_ID" \
    --data-source-id "$DATA_SOURCE_ID" \
    --region "$AWS_REGION" \
    --max-results 10 \
    --query "ingestionJobSummaries[?status=='STARTING' || status=='IN_PROGRESS'].ingestionJobId | [0]" \
    --output text) || return 1

  if [[ -n "$ACTIVE_JOB_ID" && "$ACTIVE_JOB_ID" != "None" ]]; then
    INGESTION_JOB_ID="$ACTIVE_JOB_ID"

    echo "An ingestion job is already running."
    echo "Using existing job: $INGESTION_JOB_ID"
  else
    echo "Starting a new ingestion job..."

    INGESTION_JOB_ID=$(aws bedrock-agent start-ingestion-job \
      --knowledge-base-id "$KB_ID" \
      --data-source-id "$DATA_SOURCE_ID" \
      --description "Sync OpsAI approved knowledge documents" \
      --region "$AWS_REGION" \
      --query 'ingestionJob.ingestionJobId' \
      --output text) || return 1
  fi

  if [[ -z "$INGESTION_JOB_ID" ||
        "$INGESTION_JOB_ID" == "None" ]]
  then
    echo "ERROR: An ingestion job ID was not returned."
    return 1
  fi

  update_env_value \
    "BEDROCK_INGESTION_JOB_ID" \
    "$INGESTION_JOB_ID"

  export BEDROCK_INGESTION_JOB_ID="$INGESTION_JOB_ID"

  echo
  echo "Ingestion Job ID: $INGESTION_JOB_ID"
  echo "Waiting for Knowledge Base synchronization..."

  while true
  do
    STATUS=$(aws bedrock-agent get-ingestion-job \
      --knowledge-base-id "$KB_ID" \
      --data-source-id "$DATA_SOURCE_ID" \
      --ingestion-job-id "$INGESTION_JOB_ID" \
      --region "$AWS_REGION" \
      --query 'ingestionJob.status' \
      --output text) || return 1

    echo "Sync status: $STATUS"

    case "$STATUS" in
      COMPLETE)
        echo "Knowledge Base synchronization completed successfully."
        break
        ;;

      FAILED|STOPPED)
        echo "ERROR: Knowledge Base synchronization failed."

        aws bedrock-agent get-ingestion-job \
          --knowledge-base-id "$KB_ID" \
          --data-source-id "$DATA_SOURCE_ID" \
          --ingestion-job-id "$INGESTION_JOB_ID" \
          --region "$AWS_REGION" \
          --query 'ingestionJob.failureReasons' \
          --output json

        return 1
        ;;

      STARTING|IN_PROGRESS|STOPPING)
        sleep 10
        ;;

      ""|None)
        echo "ERROR: The ingestion status could not be retrieved."
        return 1
        ;;

      *)
        echo "Waiting. Current status: $STATUS"
        sleep 10
        ;;
    esac
  done

  echo
  echo "Final ingestion result:"

  aws bedrock-agent get-ingestion-job \
    --knowledge-base-id "$KB_ID" \
    --data-source-id "$DATA_SOURCE_ID" \
    --ingestion-job-id "$INGESTION_JOB_ID" \
    --region "$AWS_REGION" \
    --query 'ingestionJob.{Status:status,Statistics:statistics,FailureReasons:failureReasons}' \
    --output json || return 1

  echo
  echo "Saved OpsAI configuration:"
  echo "  Bucket: $OPSAI_BUCKET"
  echo "  Knowledge Base ID: $KB_ID"
  echo "  Data Source Name: $DATA_SOURCE_NAME"
  echo "  Data Source ID: $DATA_SOURCE_ID"
  echo "  Ingestion Job ID: $INGESTION_JOB_ID"
}

cd ~/opsai-assistant-aws

python3 - <<'PY'
from pathlib import Path

app_file = Path("app/web/app.js")
content = app_file.read_text(encoding="utf-8")

old_code = '''function renderMarkdown(markdown) {
    const lines = String(markdown ?? "").replace(/\\r/g, "").split("\\n");
'''

new_code = r'''function normalizeModelOutput(value) {
    let text = String(value ?? "");

    // Convert HTML code blocks to Markdown code blocks.
    text = text.replace(
        /<pre[^>]*>\s*<code[^>]*>([\s\S]*?)<\/code>\s*<\/pre>/gi,
        (_, code) => `\`\`\`\n${code.trim()}\n\`\`\``
    );

    // Convert HTML headings to Markdown headings.
    text = text.replace(
        /<h([1-6])[^>]*>([\s\S]*?)<\/h\1>/gi,
        (_, level, heading) =>
            `${"#".repeat(Math.min(Number(level), 3))} ${heading.trim()}\n`
    );

    // Convert ordered HTML lists to numbered Markdown lists.
    text = text.replace(
        /<ol[^>]*>([\s\S]*?)<\/ol>/gi,
        (_, body) => {
            let number = 0;

            return body.replace(
                /<li[^>]*>([\s\S]*?)<\/li>/gi,
                (_, item) => `${++number}. ${item.trim()}\n`
            );
        }
    );

    // Convert unordered HTML lists to Markdown bullet lists.
    text = text.replace(
        /<ul[^>]*>([\s\S]*?)<\/ul>/gi,
        (_, body) =>
            body.replace(
                /<li[^>]*>([\s\S]*?)<\/li>/gi,
                (_, item) => `- ${item.trim()}\n`
            )
    );

    // Convert basic formatting tags.
    text = text.replace(
        /<strong[^>]*>([\s\S]*?)<\/strong>/gi,
        "**$1**"
    );

    text = text.replace(
        /<b[^>]*>([\s\S]*?)<\/b>/gi,
        "**$1**"
    );

    text = text.replace(
        /<em[^>]*>([\s\S]*?)<\/em>/gi,
        "*$1*"
    );

    text = text.replace(
        /<i[^>]*>([\s\S]*?)<\/i>/gi,
        "*$1*"
    );

    text = text.replace(
        /<code[^>]*>([\s\S]*?)<\/code>/gi,
        "`$1`"
    );

    text = text.replace(/<br\s*\/?>/gi, "\n");

    text = text.replace(
        /<p[^>]*>([\s\S]*?)<\/p>/gi,
        "$1\n"
    );

    // Remove remaining common layout tags.
    text = text.replace(
        /<\/?(?:div|span|section|article|header|footer|html|body)[^>]*>/gi,
        ""
    );

    return text.replace(/\n{3,}/g, "\n\n").trim();
}

function renderMarkdown(markdown) {
    const normalized = normalizeModelOutput(markdown);
    const lines = normalized.replace(/\r/g, "").split("\n");
'''

if new_code in content:
    print("HTML normalization is already configured.")
elif old_code in content:
    content = content.replace(old_code, new_code)
    app_file.write_text(content, encoding="utf-8")
    print("HTML normalization added successfully.")
else:
    raise SystemExit(
        "Could not find renderMarkdown. Do not continue with deployment."
    )
PY
